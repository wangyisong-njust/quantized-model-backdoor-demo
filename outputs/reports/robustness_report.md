# Robustness Report: DeiT-Tiny + RTMDet-Tiny

**Research question:** Does INT8 PTQ change adversarial robustness?

## Detection (RTMDet-Tiny, COCO val2017, 200 images)

> Attack: DPatch (80×80 px, 300 steps). Applied to [0,1] images before model normalization. Patch optimized on FP32, transferred to FP16/INT8.

## Key Findings

- **Classification ASR**: FP32=8.3%, FP16=8.3%, INT8=9.6% (Δ INT8-FP32=+1.3%)
- **Cls Latency**: FP32 GPU=3.5ms, INT8 CPU(ORT)=26.6ms
- **Detection DPatch vanishing rate**: FP32=4.5%, FP16=4.2%, INT8=4.3% (Δ INT8-FP32=-0.2%)
- **Det Latency**: FP32=43.1ms, FP16=44.1ms per image

## Notes

- FP16 detection: backbone+neck run in FP16; bbox_head kept FP32 (mmcv.ops.nms requires float32)
- INT8 detection: PyTorch dynamic quantization on backbone Conv2d layers only; static INT8 for full RTMDet requires mmdeploy
- INT8 classification: ONNX Runtime static quantization (QDQ format) with MinMax calibration
